# Woz Monitor (wozmon) 使い方メモ

Apple-1 に搭載された Steve Wozniak 作の小さなモニタプログラム（256 バイト、$FF00–$FFFF）。

## 起動

シミュレータを開くと自動的に起動し、プロンプト `\` が表示される。

```text
\
```

## 基本コマンド

### メモリ検査（1 アドレス）

```text
XXXX
```

例：

```text
\ FF00
FF00: D8
```

### メモリ範囲ダンプ

```text
XXXX.YYYY
```

例：

```text
\ FF00.FF0F
FF00: D8 58 A0 7F 8C 12 D0 A9
FF08: A7 8D 11 D0 8D 13 D0 C9
```

### メモリ書き込み

```text
XXXX: DD DD DD ...
```

例（$0300 に LDA #$55 / JMP $0300 を書く）：

```text
\ 0300: A9 55 4C 00 03
```

- アドレス直後にコロン（`:`）と値を入力
- スペース区切りで複数バイト続けて書ける
- Enter で確定

### プログラム実行

```text
XXXXR
```

例：

```text
\ 0300R
```

- 指定アドレスから `JMP $XXXX` 相当で実行
- プログラム内で `RTS` は使えない（スタックにリターンアドレスがない）
- `BRK` を使うとモニタに戻れる

## 実用例

### メモリに小さなプログラムを打ち込んで実行

```text
\ 0300: A9 55 85 00 A9 AA 85 01 00
\ 0300.0308
0300: A9
0301: 55
0302: 85
0303: 00
0304: A9
0305: AA
0306: 85
0307: 01
0308: 00
\ 0300R
```

上記は A=$55 → $00 番地, A=$AA → $01 番地 に書いて BRK で戻るプログラム。

### ゼロページ確認

```text
\ 0000.00FF
```

## 文字コード

- Apple-1 は ASCII の bit7 を 1 にした「高ビット ASCII」を使用
- キーボード入力は小文字 → 大文字に自動変換（シミュレータ側で処理）
- 表示は `$20`–`$7E` の範囲（bit7 は無視）

## 注意事項

| 項目 | 内容 |
| --- | --- |
| アドレス表記 | 大文字 16 進数 4 桁（`0300` など） |
| 数値表記 | 16 進数のみ（`0x` プレフィックス不要） |
| Enter | 改行兼コマンド確定 |
| バックスペース | `_`（アンダースコア）キー |
| Escape | 入力行をキャンセルして `\` に戻る |

## ROM レイアウト（参考）

| アドレス | 内容 |
| --- | --- |
| `$FF00` | RESET エントリ / NOTCR ループ |
| `$FF1A` | `\` プロンプト出力 + GETLINE |
| `$FF29` | GETCHAR（KBD ストローブ待ち） |
| `$FF3A` | SETSTOR（`:` 書き込みモード） |
| `$FF4A` | SETADR（アドレス解析） |
| `$FFDC` | PRBYTE（1 バイトを HEX 2 桁で表示） |
| `$FFE5` | PRHEX（1 ニブルを HEX 1 桁で表示） |
| `$FFEF` | ECHO（DSP への 1 文字出力） |
| `$FFFC` | リセットベクタ = `$00 $FF` → `$FF00` |
| `$FFFE` | IRQ ベクタ = `$00 $00` → `$0000` |
