# 実装計画：03-uart

## 位置づけ

Tier 1 第 3 弾。**Logic Analyzer ビューの主役**となる題材。
UART の 1 フレーム（スタートビット＋8 データビット＋ストップビット）が
Logic Analyzer 上でそのまま見える体験を作る。

---

## 仕様

### モジュール構成

```text
uart_top (ループバック統合トップ)
  ├── uart_tx  — 送信機
  └── uart_rx  — 受信機
        txd ──────────────▶ rxd  (内部ループバック)
```

### パラメータ

| パラメータ | デフォルト | 意味 |
| --- | --- | --- |
| `CLKS_PER_BIT` | 16 | 1 ビット幅 (クロック数)。シミュ用に小さく設定 |

実機(115200 baud / 100 MHz)なら 868 だが、ブラウザ可視化では 16 を使う。

---

### uart_tx — 送信機

#### ポート

| 信号 | 方向 | 意味 |
| --- | --- | --- |
| `clk` | in | クロック |
| `rst` | in | 同期リセット |
| `tx_data[7:0]` | in | 送信バイト |
| `tx_valid` | in | 送信要求（1 クロックパルス） |
| `txd` | out | シリアル送信線（アイドル=1） |
| `tx_ready` | out | 送信可能 (IDLE 状態) |
| `o_tx_state[1:0]` | out | 観測用：TX 状態 |
| `o_tx_bit[2:0]` | out | 観測用：現在送信中のビット番号 |

#### 状態機械

```text
IDLE  (2'd0) : txd=1, tx_ready=1
START (2'd1) : txd=0, CLKS_PER_BIT サイクル保持
DATA  (2'd2) : txd=data[bit], LSB first, 8ビット順送出
STOP  (2'd3) : txd=1, CLKS_PER_BIT サイクル保持 → IDLE
```

---

### uart_rx — 受信機

#### ポート

| 信号 | 方向 | 意味 |
| --- | --- | --- |
| `clk` | in | クロック |
| `rst` | in | 同期リセット |
| `rxd` | in | シリアル受信線 |
| `rx_data[7:0]` | out | 受信バイト |
| `rx_valid` | out | 受信完了パルス（1 クロック） |
| `rx_error` | out | フレーミングエラー（ストップビット=0） |
| `o_rx_state[1:0]` | out | 観測用：RX 状態 |
| `o_rx_bit[2:0]` | out | 観測用：現在受信中のビット番号 |

#### 状態機械

```text
IDLE  (2'd0) : rxd の立下りエッジ待ち
START (2'd1) : CLKS_PER_BIT/2 後にスタートビット確認（中央サンプリング）
DATA  (2'd2) : CLKS_PER_BIT ごとにサンプリング、8ビット収集
STOP  (2'd3) : ストップビット確認 → rx_valid 発行 or rx_error
```

---

### UART フレーム構造

```text
アイドル  スタート  D0 D1 D2 D3 D4 D5 D6 D7  ストップ  アイドル
  ─┐    ┌──────┐                              ┌──────┐  ┌─
   │    │  Low  │  LSB               MSB  │  High  │
   └────┘       └──────────────────────────┘        └──
```

例：'A'(0x41 = 0b01000001)の送出

```
IDLE=1, START=0, D0=1, D1=0, D2=0, D3=0, D4=0, D5=0, D6=1, D7=0, STOP=1
```

---

### ring buffer ビットレイアウト (1 サンプル = 1 uint32_t)

| ビット | 信号 | 説明 |
| --- | --- | --- |
| `[0]` | `txd` | 送信線 |
| `[1]` | `rxd` | 受信線 |
| `[2]` | `tx_valid` | 送信要求 |
| `[3]` | `tx_ready` | 送信可能 |
| `[4]` | `rx_valid` | 受信完了パルス |
| `[5]` | `rx_error` | フレーミングエラー |
| `[7:6]` | `tx_state` | TX 状態 (2bit) |
| `[9:8]` | `rx_state` | RX 状態 (2bit) |
| `[12:10]` | `tx_bit` | TX ビット番号 (3bit) |
| `[15:13]` | `rx_bit` | RX ビット番号 (3bit) |
| `[23:16]` | `tx_data` | 送信バイト (8bit) |
| `[31:24]` | `rx_data` | 受信バイト (8bit) |

## Verilog 設計方針

- `uart_tx.v` / `uart_rx.v` は独立モジュール（単体 lint 可）
- `uart_top.v` でループバック接続し、観測用出力を引き出す
- `tx_valid` は JS から 1 クロックパルスとして入力
- ビットカウンタ・クロックカウンタはリセット時に 0 クリア

```verilog
module uart_top #(
    parameter CLKS_PER_BIT = 16
) (
    input        clk, rst,
    input  [7:0] tx_data,
    input        tx_valid,
    output       tx_ready,
    // 観測用
    output [7:0] o_rx_data,
    output       o_rx_valid,
    output       o_rx_error,
    output [1:0] o_tx_state,
    output [1:0] o_rx_state,
    output [2:0] o_tx_bit,
    output [2:0] o_rx_bit,
    output       o_txd,
    output       o_rxd
);
    wire txd;
    assign o_txd = txd;
    assign o_rxd = txd; // ループバック

    uart_tx #(.CLKS_PER_BIT(CLKS_PER_BIT)) u_tx (
        .clk(clk), .rst(rst),
        .tx_data(tx_data), .tx_valid(tx_valid),
        .txd(txd), .tx_ready(tx_ready),
        .o_tx_state(o_tx_state), .o_tx_bit(o_tx_bit)
    );
    uart_rx #(.CLKS_PER_BIT(CLKS_PER_BIT)) u_rx (
        .clk(clk), .rst(rst),
        .rxd(txd),  // ループバック
        .rx_data(o_rx_data), .rx_valid(o_rx_valid), .rx_error(o_rx_error),
        .o_rx_state(o_rx_state), .o_rx_bit(o_rx_bit)
    );
endmodule
```

## 技術的なポイント・注意点

### 中央サンプリング（RX）

RXD の立下りエッジ検出後、`CLKS_PER_BIT/2` サイクル待ってスタートビット中央を確認する。
以降は `CLKS_PER_BIT` ごとにサンプリング。これにより微小なクロック誤差に耐性を持つ。

### `tx_valid` の扱い

`tx_valid` は 1 クロックのパルスを期待する設計。
JS 側は `send_byte()` を呼び出した次の `step()` で `tx_valid=1` にし、その後すぐ 0 に戻す。
連続送信時は `tx_ready=1` を確認してから次バイトを投入する。

### ループバック の可視化価値

TX の `txd` を RX の `rxd` に直結することで、同一 LA 上に送受信が同時に見える。
`CLKS_PER_BIT=16` では 1 フレーム = 160 サイクル → 1 文字分が LA の約 160px に収まる。

### bus type 拡張（LA）

02 で実装した bus type は数値の高さ表現だった。
03 では**16 進数ラベル**を状態セグメントのように表示する `hex` type を追加する方針。
