# 53. 6502 機械語入門 — Woz Monitor で動かして学ぶ

Woz Monitor に機械語を直接書き込んで実行しながら、6502 の動作を体験的に理解する。
各例題は soft-FPGA の Logic Analyzer・Register Panel で内部信号を観測できる構成になっている。

---

## 1. Woz Monitor への入力方法

```text
XXXX: AA BB CC ...    ← アドレス : バイト列  で書き込み
XXXX.YYYY             ← アドレス範囲ダンプ（読み返し確認）
XXXXR                 ← アドレスから実行
```

例題は断りなければ `$0300` に書き込んで `0300R` で起動する。
無限ループで繰り返すので Logic Analyzer を止める場合はリセットする。

## 3. 基礎編

### 3.1 LDA ループ（5 bytes）— フェッチサイクルを見る

```text
0300: A9 55 4C 00 03
```

```asm
LDA #$55      ; A ← $55（即値）
JMP $0300     ; $0300 へ戻る
```

**何が見える？**

- Register Panel: A が常に `$55`
- Logic Analyzer: `sync` が 2 クロックおきに点灯する（1 命令 = 1 フェッチ + 実行サイクル）
- Logic Analyzer: `addr` が `$0300 → $0301 → $0300 …` と繰り返す

> `sync=1` のサイクルが「命令フェッチ」、それ以外がオペランド取得や実行サイクル。
> この最小例で 6502 のパイプラインなし逐次実行を確認できる。

---

### 3.2 メモリ Read / Write（9 bytes）— WE 信号を見る

```text
0300: A9 AA 85 10 A5 10 4C 00 03
```

```asm
LDA #$AA
STA $10       ; WE=1、$0010 へ書き込み
LDA $10       ; WE=0、$0010 から読み出し
JMP $0300
```

**何が見える？**

- Logic Analyzer: `we=1` の赤セグメントが `$0010` に現れる（STA）
- Logic Analyzer: 次の LDA で同アドレスを `we=0` で読み返す
- Register Panel: A が `$AA → $AA`（書いたものが読める）

> ゼロページ（`$0000–$00FF`）は 1 バイトアドレスで指定でき、1 サイクル短い（ZP アクセスは 3 cycle、ABS は 4 cycle）。

---

### 3.3 フラグと条件分岐（11 bytes）— フラグの立ち方を見る

```text
0300: A9 FF 69 01 B0 03 4C 10 03 4C 00 03
```

```asm
         LDA #$FF
         ADC #$01   ; A=$00、C=1、Z=1、N=0
         BCS $030C  ; C=1 なので分岐する → $030C（JMP $0300）
         JMP $0310  ; ここには来ない（来たら実装バグ）
$030C:   JMP $0300
```

**何が見える？**

- Register Panel: Flags の `C=1, Z=1` を確認
- Logic Analyzer: `BCS` の分岐あり / なしでサイクル数が変わる
  - 分岐なし: 2 cycle、同ページ分岐: 3 cycle、ページクロス分岐: 4 cycle
- `$FF + $01 = $100` → 下位 8bit = $00、キャリーが立つことを確認

---

## 4. スタックとサブルーチン

### 4.1 スタック Push / Pop（11 bytes）— SP の変化を見る

```text
0300: A9 11 48 A9 22 48 68 68 4C 00 03
```

```asm
LDA #$11
PHA          ; SP: FF→FE、$01FF に書き込み
LDA #$22
PHA          ; SP: FE→FD、$01FE に書き込み
PLA          ; SP: FD→FE、$01FE から読み出し（A=$22）
PLA          ; SP: FE→FF、$01FF から読み出し（A=$11）
JMP $0300
```

**何が見える？**

- Register Panel: SP が `FF → FE → FD → FE → FF` と変化する
- Logic Analyzer: `$01FF`・`$01FE` への `we=1` / `we=0` が交互に現れる
- **LIFO**（後入れ先出し）の動作を信号レベルで確認できる

> 6502 のスタックは `$0100–$01FF` に固定。SP は先デクリメント（push 前に SP-1 してから書く）。

---

### 4.2 JSR / RTS — サブルーチン呼び出し（12 bytes）

```text
0300: 20 08 03 20 08 03 4C 00 03 E8 60
```

```asm
$0300: JSR $0308    ; PC+2 をスタックに push（2 bytes）して $0308 へ
$0303: JSR $0308    ; 2 回目
$0306: JMP $0300
$0308: INX          ; X をインクリメント
$0309: RTS          ; スタックから戻りアドレスを pop して +1 して PC へ
```

**何が見える？**

- Register Panel: X が呼び出しごとに +1（`$01 → $02 → …`）
- Logic Analyzer: JSR で `$01FE/$01FF` への write、RTS で read が見える
- Logic Analyzer: JSR は 6 cycle、RTS は 6 cycle（`sync` 間隔が長い）

> RTS は pop したアドレス +1 を PC にセットする（JSR は PC+2 の -1 = PC+1 を push するため）。

---

## 5. アドレッシングモード

### 5.1 絶対 X インデックス（18 bytes）— 配列アクセスの仕組み

まずデータを配置してから実行する:

```text
0320: 11 22 33 44 55
0300: A2 00 BD 20 03 E8 E0 05 D0 F8 4C 00 03
```

```asm
       LDX #$00
loop:  LDA $0320,X  ; $0320 + X のアドレスを読む
       INX
       CPX #$05
       BNE loop
       JMP $0300
```

**何が見える？**

- Logic Analyzer: `addr` が `$0320 → $0321 → $0322 → $0323 → $0324` と順に現れる
- Register Panel: X が 0 → 1 → 2 → 3 → 4 → 0 とカウントする
- `LDA abs,X` は通常 4 cycle、ページクロス時は 5 cycle（例題 9 参照）

---

### 5.2 ゼロページ X インデックス（12 bytes）— ZP の速度優位

まずデータを配置:

```text
0050: 11 22 33 44
0300: A2 00 B5 50 E8 E0 04 D0 F9 4C 00 03
```

```asm
       LDX #$00
loop:  LDA $50,X    ; ZP + X でアクセス（$50〜$53 を順に読む）
       INX
       CPX #$04
       BNE loop
       JMP $0300
```

**何が見える？**

- Logic Analyzer: アドレスが `$0050 → $0051 → $0052 → $0053` と ZP 内で移動する
- `LDA zp,X` は 4 cycle（vs `LDA abs,X` の 4〜5 cycle）で同等だが、**命令が 1 バイト短い**（2 bytes vs 3 bytes）

---

### 5.3 間接インデックス (zp),Y — 2 段ポインタ（14 bytes）

まずポインタとバッファを配置:

```text
0020: 00 04     ← ポインタ（リトルエンディアンで $0400 を指す）
0300: A0 00 A9 00 91 20 C8 C0 04 D0 F9 4C 00 03
```

```asm
       LDY #$00
       LDA #$00
loop:  STA ($20),Y  ; ZP[$20/$21]=$0400、+Y → $0400+Y に書き込み
       INY
       CPY #$04
       BNE loop
       JMP $0300
```

**何が見える？**

- Logic Analyzer: ZP 読み出し（`$0020`・`$0021`）→ ポインタ先 `$0400+Y` への書き込みという**2 段アクセス**が見える
- `STA (zp),Y` は 6 cycle（ZP 2 回読み + アドレス加算 + write）

> このモードは C 言語のポインタ経由書き込み `*(ptr + y) = val` に対応する。

## 6. ビット操作

### 6.1 論理演算 AND / ORA / EOR（12 bytes）— マスク・セット・トグル

```text
0300: A9 FF 29 0F 09 F0 49 AA 4C 00 03
```

```asm
LDA #$FF
AND #$0F     ; A=$0F — 下位 4bit だけ残す（マスク）
ORA #$F0     ; A=$FF — 上位 4bit をセット
EOR #$AA     ; A=$55 — ビット反転（XOR）
JMP $0300
```

**何が見える？**

- Register Panel: A が `$FF → $0F → $FF → $55` と変化する
- Register Panel: N フラグが `1 → 0 → 1 → 0` と変化する（bit7 に連動）

| 演算 | 用途例 |
|------|--------|
| `AND` | フラグビットのクリア・マスク |
| `ORA` | フラグビットのセット |
| `EOR` | フラグビットのトグル、簡易暗号化 |

---

### 6.2 シフト・ローテート（15 bytes）— キャリーとの連動

```text
0300: A9 01 0A 0A 0A 4A 38 2A 6A 4C 00 03
```

```asm
LDA #$01
ASL A        ; A=$02  C=0（×2）
ASL A        ; A=$04  C=0
ASL A        ; A=$08  C=0
LSR A        ; A=$04  C=0（÷2）
SEC          ; C=1 をセット
ROL A        ; A=$09  C=0（左ローテート、bit0 に C が入る）
ROR A        ; A=$04  C=1（右ローテート、bit7 に C が入る）
JMP $0300
```

**何が見える？**

- Register Panel: A のビットパターンと C フラグの連動を 1 step ずつ確認できる
- `ASL`/`LSR` は bit7/bit0 が C へ逃げる、`ROL`/`ROR` は C と端ビットが入れ替わる

> 多バイト整数の左右シフトは ROL/ROR を上位バイト → 下位バイトの順で連鎖させて実装する。

## 7. サイクル精度

### 7.1 ページクロスペナルティ（13 bytes）— +1 サイクルを見る

まずデータを配置:

```text
03FF: 42
0300: A2 01 BD FE 02 A0 01 B9 FE 02 4C 00 03
```

```asm
LDX #$01
LDA $02FF,X  ; $02FF + 1 = $0300（ページクロス） → 5 cycle
LDY #$01
LDA $02FF,Y  ; 同上 → 5 cycle
JMP $0300
```

**何が見える？**

- Logic Analyzer: `LDA abs,X` がページクロス時に 5 cycle になる（`sync` 間隔が通常より 1 伸びる）
- Logic Analyzer: ページ誤読み（`$02FF` への余分なアクセス）が見える（実チップの挙動を再現）

> ページクロスとは上位アドレスバイト（A8〜A15）が変化すること（例: `$02FF → $0300`）。
> 6502 はアドレス加算の結果が別ページに入ると、誤ったアドレスで 1 サイクル余分に読む。

---

### 7.2 RMW 命令 INC / DEC（11 bytes）— Read-Modify-Write を見る

```text
0300: A9 00 85 20 E6 20 E6 20 C6 20 4C 00 03
```

```asm
LDA #$00
STA $20      ; $20 = $00 に初期化
INC $20      ; $20 = $01（Read → Modify → Write の 5 cycle）
INC $20      ; $20 = $02
DEC $20      ; $20 = $01
JMP $0300
```

**何が見える？**

- Logic Analyzer: `INC` 1 命令で `addr=$0020` への **read → write** が連続して見える
- Logic Analyzer: write 直前に同アドレスへの無効 write が 1 サイクル挟まる（実チップ挙動）

> RMW（Read-Modify-Write）命令は INC / DEC / ASL / LSR / ROL / ROR のメモリ対象版。
> 内部で「読む → 元値を書く（無効） → 新値を書く」の 3 フェーズが見える。
> 古いハードウェア（DMA 等）との干渉でバグになることがある有名な挙動。
