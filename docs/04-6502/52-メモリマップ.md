# 52. Apple-1 メモリマップ（Woz Monitor + Integer BASIC）

## 1. 全体マップ（64KB アドレス空間）

```text
アドレス           サイズ    内容
─────────────────────────────────────────────────────
$0000–$00FF        256 B   ゼロページ（6502 高速アクセス）
$0100–$01FF        256 B   6502 ハードウェアスタック
$0200–$027F        128 B   Woz Monitor 入力バッファ (GETLINE)
$0280–$0FFF       ~3.5 KB  BASIC プログラム領域・変数（RAM）
$1000–$CFFF         ─      未実装（アクセス時は未定義）
$D010–$D013          4 B   PIA（6821）— キーボード・ディスプレイ I/O
$D014–$DFFF         ─      PIA ミラー / 未使用
$E000–$EFFF        4 KB    Integer BASIC ROM
$F000–$FEFF         ─      未実装
$FF00–$FFFF        256 B   Woz Monitor ROM
```

> 標準 Apple-1 の RAM は $0000–$0FFF の 4 KB（拡張ボードで最大 8 KB）。

---

## 2. I/O — PIA（6821）レジスタ

| アドレス | 名称 | 方向 | 説明 |
|---------|------|------|------|
| `$D010` | KBD | R | キーボードデータ（bit7=1 で新しいキー） |
| `$D011` | KBDCR | R/W | キーボード制御レジスタ（bit7=キー ready フラグ） |
| `$D012` | DSP | W | 表示データ（bit7=1 で書き込みトリガ） |
| `$D013` | DSPCR | R/W | ディスプレイ制御レジスタ |

- キーボード: PIA の CA1 に strobe → KBDCR.bit7 が立つ → KBD を読む → フラグがクリアされる
- ディスプレイ: DSP に ASCII を書く（bit7 セット）→ PIA が 7-seg / TV 出力へ転送

---

## 3. 6502 ベクタ（ROM 末尾）

| アドレス | ベクタ | 指す先 |
|---------|--------|--------|
| `$FFFA–$FFFB` | NMI | （Apple-1 では未使用） |
| `$FFFC–$FFFD` | RESET | `$FF00`（Woz Monitor エントリ） |
| `$FFFE–$FFFF` | IRQ/BRK | `$FF00` または Monitor ハンドラ |

---

## 4. Woz Monitor（$FF00–$FFFF）

### 4.1 主要エントリポイント

| アドレス | ラベル | 機能 |
|---------|--------|------|
| `$FF00` | RESET | モニタ起動・初期化 |
| `$FFEF` | ECHO | 1 文字出力（A レジスタ → DSP） |
| `$FFDC` | PRHEX | A の下位 4 bit を HEX 1 桁出力 |
| `$FFE5` | PRBYTE | A を HEX 2 桁出力 |
| `$FFEB` | PRADR | XAMH:XAML を "XXXX:" 形式で出力 |

> アドレスは Woz Monitor ソース（Apple-1 Operation Manual 収録）に基づく。
> 移植版・FPGA 実装では若干異なる場合がある。

### 4.2 ゼロページ変数（$24–$2B）

| アドレス | ラベル | 説明 |
|---------|--------|------|
| `$24` | XAML | examine/store の低バイト |
| `$25` | XAMH | examine/store の高バイト |
| `$26` | STL | ストア先アドレス低バイト |
| `$27` | STH | ストア先アドレス高バイト |
| `$28` | L | 汎用テンポラリ（低） |
| `$29` | H | 汎用テンポラリ（高） |
| `$2A` | YSAV | Y レジスタ退避 |
| `$2B` | MODE | 動作モード（`$00`=XAMMODE / `$EE`=STOREMODE） |

### 4.3 入力バッファ

- `$0200–$027F`（128 B）: `GETLINE` で 1 行分を受け取るバッファ

---

## 5. Integer BASIC ROM（$E000–$EFFF）

### 5.1 主要エントリポイント

| アドレス | 機能 |
|---------|------|
| `$E000` | BASIC コールドスタート（NEW 相当、全初期化） |
| `$E2B3` | BASIC ウォームスタート（プログラム保持したまま再起動）※ |

> ※ ウォームスタートアドレスは実装により異なる場合がある。要ソース確認。

### 5.2 BASIC が使うゼロページ（抜粋）

BASIC は $00–$4F 付近を広く使用する。主なポインタ:

| アドレス | 用途 |
|---------|------|
| `$00–$01` | プログラムカウンタ（テキストポインタ） |
| `$02–$03` | LOMEM（プログラム先頭） |
| `$04–$05` | HIMEM（変数領域上端） |
| `$06–$07` | 変数テーブル先頭 |
| `$08–$09` | FOR-NEXT スタックポインタ |
| `$3A–$3B` | 現在の行番号 |

> 詳細は Apple-1 BASIC ソースリスト（Woz 直筆）参照。
> ゼロページは密に使われており、Woz Monitor と競合しないよう $24–$2B は BASIC が避けている。

### 5.3 BASIC メモリレイアウト（実行時）

```text
$0280          LOMEM（プログラムテキスト開始）
  ↓
  プログラムテキスト
  （各行: 行番号2B + 長さ1B + トークン列 + $00）
  ↓
  変数テーブル（名前2B + 値2B = 4B/変数）
  ↓
  配列テーブル
  ↓
  空き領域
  ↑
$0FFF          HIMEM（4KB RAM 末端）
```

- プログラム＋変数＋配列の合計上限: 約 500 B（詳細は §11 of [51-integer-BASIC-文法リファレンス.md](51-integer-BASIC-文法リファレンス.md)）

## 6. アドレスデコーダ設計への示唆

soft-FPGA で Apple-1 を実装する場合の最低限のデコード:

| 選択条件 | チップ/機能 |
|----------|------------|
| A15:A12 = `0000`–`0FFF` | RAM（4 KB） |
| A15:A4 = `1101 0000 0001` (`$D01x`) | PIA |
| A15:A12 = `1110` (`$E000`–`$EFFF`) | BASIC ROM |
| A15:A8 = `1111 1111` (`$FF00`–`$FFFF`) | Woz Monitor ROM |
