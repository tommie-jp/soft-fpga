# 51. Apple-1 Integer BASIC 文法リファレンス

## 1. 基本構造

```basic
行番号 文
行番号 文 : 文 : 文    ← コロンで複文（落とし穴あり → §6）
```

- 行番号: 1〜32767
- 行番号なしで入力するとダイレクトモード（即時実行）

---

## 2. 変数

| 種別 | 形式 | 例 |
|------|------|----|
| 整数変数 | 英字 1 文字、または英字＋数字 | `A`, `B3`, `Z9` |
| 配列 | `A(N)` — 要 `DIM` 宣言 | `A(10)` |

- **変数名は最大 2 文字**（`FLAGS` と書いても `F` として解釈される）
- **整数のみ**（-32768〜32767）— 浮動小数点なし

---

## 3. 宣言

```basic
DIM A(N)    ← A(1)〜A(N) を確保
```

- 添字は **1 始まり**。`A(0)` は `*** RANGE ERR`
- **遅延確保**（Lazy allocation）— `DIM` 時点では実メモリを確保しない。
  FOR ループ内で要素を初めて書くと変数テーブルが拡張・移動し、
  FOR-NEXT スタックの変数アドレスが無効化 → `*** BAD NEXT ERR`
  - **回避策**: 配列の初期化は `GOTO` ベースのループで行い、全要素確保後に `FOR-NEXT` を使う

---

## 4. 代入

```basic
LET A = 式    ← LET は省略可
A = 式
```

---

## 5. 演算子

| 種別 | 演算子 |
|------|--------|
| 算術 | `+` `-` `*` `/` `MOD` |
| 比較 | `=` `<` `>` `<=` `>=` `<>` |
| 論理 | `AND` `OR` `NOT` |

- `/` は**整数除算**（切り捨て）
- `MOD` は剰余

---

## 6. 制御フロー

```basic
GOTO 行番号
GOSUB 行番号
RETURN
END
```

```basic
FOR 変数 = 開始 TO 終了
  ...
NEXT 変数          ← 変数名は省略不可

FOR 変数 = 開始 TO 終了 STEP 増分
```

---

## 7. IF 文

```basic
IF 条件 THEN 文
IF 条件 THEN GOTO 行番号
```

**落とし穴 — コロン後の文は常に実行される:**

```basic
IF A=1 THEN B=2 : C=3    ← C=3 は条件に関わらず常に実行
```

複数の文を条件付きにしたい場合は `GOTO` で別行に飛ぶ。

```basic
100 IF A<>1 THEN GOTO 200
110 B=2
120 C=3
200 ...
```

---

## 8. 入出力

```basic
PRINT 式
PRINT 式, 式        ← カンマ: タブ位置へ移動
PRINT 式; 式        ← セミコロン: スペースなし連結
PRINT               ← 改行のみ
PRINT "文字列"      ← 文字列リテラル（変数は非サポート）

INPUT 変数          ← プロンプトなし、整数入力
```

## 10. プログラム管理

```basic
LIST     ← プログラム一覧

NEW      ← プログラム消去
```

---

## 11. メモリ制約

- プログラムテキスト＋変数＋配列の合計 **約 500 バイト以内**
- 配列サイズの目安: `N <= 100` 程度（`DIM A(100)` = 200 バイト）
- `DIM A(999)` → `*** MEM FULL ERR`

---

## 12. エラー一覧

| エラー | 原因 |
|--------|------|
| `*** RANGE ERR` | 配列添字が範囲外（0 含む） |
| `*** MEM FULL ERR` | メモリ不足 |
| `*** BAD NEXT ERR` | FOR-NEXT スタック破壊（DIM 遅延確保が原因のことが多い） |
| `*** SYNTAX ERR` | 構文エラー |
| `*** UNDEF'D STATEMENT` | 存在しない行番号への GOTO/GOSUB |
