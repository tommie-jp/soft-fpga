# 13. テスト結果・ベンチマーク記録

Verilator ホストシミュレーション（Docker コンテナ内）での実行結果。
WebAssembly / Pico 2 実行時との比較用ベースライン。

実行日: 2026-05-13  
実行環境: WSL2 (Linux 6.6.87.2-microsoft-standard-WSL2)  
コマンド: `./doTestAll.sh`

## ホスト CPU スペック

| 項目 | 値 |
| --- | --- |
| CPU | Intel Core Ultra 9 285K |
| TSC 実測周波数 | **3.686 GHz**（TSC delta 3,686,826,852 / 1.000 s） |
| BogoMIPS | 7372.80（Linux カーネルの busy-wait 較正値 ≈ TSC周波数 × 2） |
| コア数 | 24 コア（SMT 無効） |

> **BogoMIPS は WebAssembly / Pico 2 との比較には使えない。**  
> BogoMIPS はホスト CPU の busy-wait ループ速度であり、Verilator シミュレーションの  
> スループットとは無関係。比較に意味があるのは「シミュレートされた 6502 クロック数 ÷ 壁時計時間」  
> すなわち **実効シミュレーション MHz** のみ。

## Verilator 実効シミュレーション速度（Dormann テストより）

| 環境 | 実測値 |
| --- | --- |
| Verilator on WSL2 (Core Ultra 9 285K) | **≈ 13.7 MHz 相当** |
| 実機 Apple-1 クロック | 1.023 MHz |
| 比率（Verilator / 実機） | ≈ 13× 速 |
| ホスト CPU 実クロック比 | ≈ 1/270（3.686 GHz の 270 分の 1 速度） |

WebAssembly・Pico 2 でのシミュレーション速度が判明したらここに追記する。

## 5. Klaus Dormann 6502 Functional Test

`verif/dormann/test_dormann 6502_functional_test.bin` —  
全 6502 命令・アドレッシングモードを網羅するリグレッションテスト。  
成功条件: PC=$3469 の無限ループ（JMP *）に到達すること。

| 項目 | 値 |
| --- | --- |
| 結果 | **PASS** (trapped at $3469) |
| cycles | 96,241,373 |
| @1MHz 換算 | 96,241 ms（≒96.2 秒） |
| @1.023MHz 換算 | 94,078 ms（≒94.1 秒） |
| 実時間（Verilator） | 7,042 ms（≒7.0 秒） |

> Verilator シミュレーションの実行速度は約 **13.7 MHz 相当**（96,241,373 cycles ÷ 7.042 s）。
