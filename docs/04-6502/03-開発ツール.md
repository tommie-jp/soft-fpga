# Apple-I 向け開発ツールメモ

## 1. C コンパイラ

### 1.1 cc65（推奨）

- **URL**: <https://cc65.github.io/>
- Linux で動く 6502 向けクロス開発パッケージ
- C コンパイラ・アセンブラ・リンカ・ライブラリが一体
- Apple-1 公式ターゲットはないが、カスタムリンカスクリプト 1 ファイルで対応できる
- インストール: `sudo apt install cc65`

```bash
cc65 -t none -O foo.c       # C → アセンブリ
ca65 foo.s -o foo.o         # アセンブリ → オブジェクト
ld65 -C apple1.cfg foo.o -o foo.bin  # リンク（カスタム cfg 要作成）
```

Apple-1 用の最小リンカスクリプト例（`apple1.cfg`）:

```text
MEMORY {
  RAM: start = $0300, size = $CD00, file = %O;
}
SEGMENTS {
  CODE: load = RAM, type = ro;
  DATA: load = RAM, type = rw;
  BSS:  load = RAM, type = bss;
}
```

#### 制約

- ANSI C のサブセット（`long long` 非対応など）
- OS・標準ライブラリなし → `printf` 等は自前実装が必要
- `putchar` 相当は DSP (`$D012`) への直接書き込みで代替

### 1.2 llvm-mos（最適化重視）

- **URL**: <https://llvm-mos.org/>
- LLVM ベースの 6502 向け C/C++ コンパイラ
- cc65 より最適化が強力（ゼロページをレジスタとして活用）
- C99 / C++11 対応（`clang` コマンドで使用）
- Apple-1 公式ターゲットなし、SDK のカスタマイズが必要
- バイナリリリースあり（Ubuntu 対応）: <https://github.com/llvm-mos/llvm-mos>

## 4. Apple-1 での C 開発の現実

Apple-1 には OS も標準ライブラリも存在しない。
C で書いても使えるのは純粋な計算ロジックのみで、I/O はすべて直接ハードウェアアクセスになる。

```c
/* Apple-1 での文字出力（cc65 / llvm-mos 共通） */
#define DSP  (*(volatile unsigned char *)0xD012)
#define DSPCR (*(volatile unsigned char *)0xD013)

void putch(char c) {
    DSP = c | 0x80;
    while (DSP & 0x80)  /* busy wait */
        ;
}
```

メモリ制約（RAM 4 KB = $0000–$0FFF）を考えると、
C よりアセンブラで直接書く方が現実的な場面が多い。
