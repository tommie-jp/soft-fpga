# 21. サイクル計測プログラム例

`sw/cpm/bench/` に置いた BDS C プログラム3本。  
`cycl.h` ([20-サイクルカウンタ.md](20-サイクルカウンタ.md)) と組み合わせて使う。

## 1. calib.c — カウンタ動作確認

**目的**: サイクルカウンタが正しく動いているかを IN 命令で確認する。

BDS C 1.60 は `#asm`/`#endasm` 非対応のため、NOP の代わりに `inp(0x10)` を使用する。  
`IN p` = 10 T-states、4 回 = 40 T（10 NOP × 4T と同等）。

```
=== Cycle Counter Calibration ===

cycl_read overhead : ~70 T
empty loop 500x    : ~12500 T  (~25 T/iter)
4x IN loop 500x    : ~32500 T  (~65 T/iter, +40 vs empty)
  => +40 T vs empty (~40 expected = 4 INs x 10T)
```

**読み方**:
- `+40 T` が出れば IN = 10T / 命令 → カウンタ正常
- BDS C の `for` ループ自体は約 25 T/iter（LXI + CMP + JNZ + INX）

## 2. instbench.c — I/O・メモリ・演算操作の計測

**目的**: 主要操作の T ステートを実測してデータシートと照合する。

BDS C 1.60 は `#asm` 非対応のため、`inp()`/`outp()` + 純 C 演算で計測する。

```
=== 8080 Operation Benchmark  N=500 ===

Loop baseline: 25 T/iter

  4x IN  p        : 40 T extra  (expected 40 = 4x10T)
  4x OUT p,A      : 40 T extra  (expected 40 = 4x10T)
  4x mem read     : ~28 T extra  (expected ~28 = 4x7T MOV A,M)
  4x mem write    : ~28 T extra  (expected ~28 = 4x7T MOV M,A)
  8x int add      : ~32 T extra  (expected ~32 = 8x4T ADD r)
  C function call : ~27 T extra  (expected ~27 = CALL+RET)
```

**読み方**: "expected" と実測が一致すれば vm80a RTL の T ステート実装が正確。

Intel 8080 データシートとの比較表:

| 操作 | 命令 | 期待値 | 実測 | 判定 |
|------|------|--------|------|------|
| `inp()` (IN) | `IN p` | 10T | — | — |
| `outp()` (OUT) | `OUT p,A` | 10T | — | — |
| `char` 読み取り | `MOV A,M` | 7T | — | — |
| `char` 書き込み | `MOV M,A` | 7T | — | — |
| `int` 加算 | `ADD r` | 4T | — | — |
| CALL+RET | `CALL nn / RET` | 27T | — | — |

（実測値を埋めて記録する）

## 3. sort.c — バブルソート vs 挿入ソート

**目的**: アルゴリズムの違いによる T ステート差を実測する。

```
=== Sort Benchmark  N=32 ===

Worst case (reverse):
  bubble sort             : ~30000 T
  insertion sort          : ~20000 T

Random data:
  bubble sort             : ~20000 T
  insertion sort          : ~13000 T
```

（上記は予想値。実測値は異なる）

**読み方**:
- 逆順（最悪ケース）で挿入ソートが約 35% 速い理由: バブルソートは必ず O(N²) の swap を実行するが、挿入ソートは比較後に配列をシフトするだけで swap のオーバーヘッドが少ない
- 8080 の場合、swap は 3 回の `MOV` = 15T に対し、シフトは 1 回の `MOV` = 7T

## コンパイル手順

```
A> CC CYCL.C
A> CC CALIB.C
A> CLINK CALIB CYCL
A> CALIB

A> CC CYCL.C
A> CC INSTBENC.C
A> CLINK INSTBENC CYCL
A> INSTBENC

A> CC CYCL.C
A> CC SORT.C
A> CLINK SORT CYCL
A> SORT
```

`cycl.h` は同じディレクトリに置くか、コンパイル前に A: にコピーする。
