# 20. サイクルカウンタ (T ステート計測)

soft-FPGA CP/M シミュレータ専用の T ステート（クロックサイクル）計測機能。  
`step()` 1 回 = 1 T ステートをハーネス内部でカウントし、I/O ポート経由で読み出す。

## 1. I/O ポート仕様

| ポート | 方向 | 動作 |
|--------|------|------|
| `30h` | OUT | カウンタをリセット (値は無視) |
| `30h` | IN  | カウンタをラッチし bits 0-7 を返す |
| `31h` | IN  | ラッチした値の bits 8-15 を返す |
| `32h` | IN  | ラッチした値の bits 16-23 を返す |
| `33h` | IN  | ラッチした値の bits 24-31 を返す |

`IN 30h` で 32 ビット値が一括ラッチされるため、4 回の `IN` は一貫した値を返す。

カウンタは 32 ビット符号なし整数。  
2MHz 実機換算で約 2000 秒でオーバーフローするが、シミュレータ速度では問題になりにくい。

## 2. BDS C からの使い方

ヘッダファイル: `sw/cpm/cycl.h`

```c
#include "cycl.h"

/* 計測例 (BDS C 1.60: long 非対応、int のみ) */
int t0, t1;

cycl_reset();
t0 = cycl_read();

/* ← ここに計測したいコード (< 65535 T) ← */
for (i = 0; i < 500; i++)
    buf[i] = buf[i] + 1;

t1 = cycl_read();
printf("cycles: %d\n", t1 - t0);
```

### 関数

| 関数 | 戻り値 | 説明 |
|------|--------|------|
| `cycl_reset()` | — | カウンタを 0 にリセット（マクロ） |
| `cycl_read()` | `int` (16 bit) | T ステート数を返す（最大 65535 T） |

**BDS C 1.60 の制約**: `long` 型非対応のため 16 ビット（0〜65535 T）に限定。  
65535 T = 実機 2MHz 換算で約 33ms。それ以上の測定は N を小さくして分割する。

```c
/* オーバーヘッド測定 */
int t0, t1;
cycl_reset();
t0 = cycl_read();
t1 = cycl_read();
/* overhead = t1 - t0  (通常 ~30-40 T) */
```

## 3. アセンブラからの使い方

```asm
; リセット
XRA A
OUT 30H

; ラッチ＆読み出し (16 ビット: BC = count)
IN  30H     ; latch, bits 0-7
MOV C, A
IN  31H     ; bits 8-15
MOV B, A
; BC = 16-bit T ステート数
```

## 4. 8080 主要命令の T ステート数（参考）

| 命令 | T ステート |
|------|-----------|
| `MOV r,r` | 5 |
| `MVI r,n` | 7 |
| `LXI rp,nn` | 10 |
| `ADD r` | 4 |
| `INR r` | 5 |
| `DCR r` | 5 |
| `JMP nn` | 10 |
| `CALL nn` | 17 |
| `RET` | 10 |
| `IN p` | 10 |
| `OUT p` | 10 |
| `LHLD nn` | 16 |
| `SHLD nn` | 16 |

## 5. 注意事項

- このカウンタは **soft-FPGA シミュレータ専用**。実機 CP/M では動作しない。
- `--ro` / `--no-save` フラグと独立して動作する。
- `sim_init()` を呼ぶとカウンタはリセットされる（CP/M 起動時に 0 から開始）。
- 非常に長い計測（> 2^32 T ステート）ではオーバーフローに注意。
