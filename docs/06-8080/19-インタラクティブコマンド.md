# 19. インタラクティブコマンド

CP/M シミュレータ (`cpm`) の対話モードで使えるキー操作と `!` コマンド。

## 1. 特殊キー

| キー | 動作 |
|------|------|
| `Ctrl+\` | シミュレータ終了（ダーティドライブを自動保存） |
| `Ctrl+T` | ステータス表示（PC・ドライブ状態） |
| `↑` / `↓` | コマンドヒストリ（前後のエントリへ移動） |

### `Ctrl+T` 表示例

```
--- [Ctrl+T] status ---
CPU   PC=F234  [BIOS]
Disk  drive=B  trk=12  sec=5  dma=0080
      A: cpm22.dsk          [dirty]
      B: bdsc.dsk
      C: (blank)
      D: (blank)
```

PC 領域の分類:

| 範囲 | 表示 |
|------|------|
| `0x0000–0x00FF` | `page0` |
| `0x0100–0xDBFF` | `TPA` |
| `0xDC00–0xE3FF` | `CCP` |
| `0xE400–0xF1FF` | `BDOS` |
| `0xF200–0xFFFF` | `BIOS` |

## 2. コマンドヒストリ

CP/M の `A>` プロンプトで ↑/↓ キーを押すと、入力したコマンドの履歴を呼び出せる。

| キー | 動作 |
|------|------|
| `↑` | 1 つ古いエントリを表示 |
| `↓` | 1 つ新しいエントリを表示。最新まで戻ると↑を押した時点の入力に戻る |

### 2.1 動作の詳細

- 現在の入力をバックスペースで消去してから履歴エントリを表示する。
- 履歴エントリを表示した状態で文字を打てばそのまま編集できる。
- `Enter` を押した行をヒストリに追加する（空行・直前と同じ内容は除外）。
- 最大 100 件保持。上限超過時は古いエントリから削除される。

### 2.2 実装上の注意

ホスト側 (`main_linux.cpp`) で ESC シーケンス (`ESC [ A` / `ESC [ B`) を横取りして
バックスペースと新エントリを `send_key()` で CP/M に送り込む。
CP/M 自体はヒストリを知らず、CCP のラインバッファ経由で自然に動作する。

## 3. `!` コマンド

行頭で `!` を入力するとコマンドモードに入る。`Enter` で実行。

### 2.1 シェルエスケープ

```
!ls
!pwd
!cp foo.com ~/backup/
```

シェルコマンドをそのまま実行し、出力をターミナルに表示する。

### 2.2 `!cap` — 出力キャプチャ

```
!cap output.txt      # キャプチャ開始
!cap off             # キャプチャ停止
!cap                 # キャプチャ停止（引数なし）
```

CP/M のコンソール出力をファイルに保存する。実行中はリアルタイムで書き込まれる。

**用途**: CPU エクササイザの結果保存、DIR 一覧の記録など。

### 2.3 `!paste` — テキスト流し込み

```
!paste program.bas
!paste commands.txt
```

テキストファイルの内容をキーボード入力として CP/M に流し込む。
`\n` は自動的に `\r` に変換される。

**用途**: MBASIC プログラムの入力、繰り返しコマンド列の自動実行。

**注意**: 最大 65,535 バイト。CP/M の入力バッファを圧迫しないよう 16 バイト/ループずつ送信する。

### 2.4 `!mount` — ディスク交換

```
!mount A: /path/to/new.dsk
!mount B: work.dsk
```

実行中の CP/M を再起動せずにドライブのディスクイメージを差し替える。
差し替え後のパスは終了時の自動保存先にも反映される。

**用途**: 作業ディスクの切り替え、別プロジェクトのディスクをマウント。

## 3. 起動フラグ（関連）

| フラグ | 動作 |
|--------|------|
| `--orig` | `.orig.dsk`（原本）から読み込み、保存先は `.dsk` |
| `--no-save` | 終了時にディスクを保存しない |
| `--ro` | 全ドライブ読み取り専用 |
