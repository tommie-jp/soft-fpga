# 15. ファイル構成とホストとのファイル交換

`cpm` コマンド（Intel 8080 / CP/M 2.2 soft-FPGA シミュレータ）が参照するファイルの場所と、
ホスト Ubuntu との間でファイルを交換する手順をまとめる。

## 1. ディレクトリ構成

```
examples/06-8080/               ← このサブプロジェクトのルート
├── CMakeLists.txt              ← ビルド定義（成果物: build/cpm）
├── cxx/
│   ├── harness.cpp             ← Verilator ハーネス・I/O ポート処理
│   └── main_linux.cpp          ← Linux エントリポイント・引数処理・パス解決
├── verilog/
│   ├── cpm_top.v               ← トップモジュール（RAM・I/O バス）
│   └── vm80a/                  ← Intel 8080 RTL（サブモジュール）
├── rom/
│   └── cpm22.bin               ← CCP + BDOS バイナリ（5632 B）
│                                  RunCPM の CPM22.SYS 由来
├── sw/cpm/
│   ├── bios/
│   │   ├── bios.asm            ← カスタム BIOS ソース（z80asm）
│   │   └── bios.bin            ← ビルド済み BIOS バイナリ
│   └── disks/
│       └── cpm22.dsk           ← CP/M ディスクイメージ（IBM 3740: 256 KB）
│                                  ED.COM / PIP.COM / DDT.COM 等を収録
├── test/
│   ├── test_all.bin            ← ベアメタル統合テストバイナリ
│   └── 8080ex1_bare.bin        ← 8080 命令エクササイザ（ベアメタル版）
└── build/                      ← cmake ビルドディレクトリ
    └── cpm                     ← 実行バイナリ
```

## 2. 起動時のパス解決

`cpm` は自バイナリのパスから **1 つ上のディレクトリ**を基準に各ファイルを探す。

```
実行バイナリ: examples/06-8080/build/cpm
基準ディレクトリ: examples/06-8080/   ← build/ の親
```

| ファイル | デフォルトパス（基準ディレクトリ相対） |
|---------|--------------------------------------|
| BIOS    | `sw/cpm/bios/bios.bin` |
| CCP+BDOS | `rom/cpm22.bin` |
| ディスクイメージ | `sw/cpm/disks/cpm22.dsk` |
| テストバイナリ | `test/test_all.bin` |

別のパスを使う場合はコマンドラインオプションで指定できる（[14-cpm-コマンド説明.md](14-cpm-コマンド説明.md) 参照）。

## 3. CP/M メモリマップ（起動時）

```
$FFFF ┬──────────────────────────────
      │  BIOS（カスタム）             ← bios.bin を $F200 に配置
$F200 ├──────────────────────────────
      │  BDOS                         ┐
$E400 ├──────────────────────────────  ├ cpm22.bin を $DC00 に配置
      │  CCP                          ┘
$DC00 ├──────────────────────────────
      │  TPA（ユーザープログラム領域）  ← .COM ファイルがここにロードされる
$0100 ├──────────────────────────────
      │  ページ 0（ベクタ・BDOS エントリ）
$0000 ┴──────────────────────────────
```

## 4. ホストとのファイル交換

### 4.1 cpmtools（現時点で確実・追加実装不要）

CP/M を**停止した状態**でホストから直接 DSK イメージを操作する。

```bash
# ホスト → CP/M ディスク
cpmcp -f ibm-3740 sw/cpm/disks/cpm22.dsk ./hello.asm 0:HELLO.ASM

# CP/M ディスク → ホスト
cpmcp -f ibm-3740 sw/cpm/disks/cpm22.dsk 0:RESULT.TXT ./result.txt

# ディスク内ファイル一覧
cpmls -f ibm-3740 sw/cpm/disks/cpm22.dsk
```

> **注意**: `cpm` 実行中に DSK ファイルを操作すると内容が壊れる可能性がある。
> 必ず `Ctrl+\` で CP/M を停止してから実行すること。

### 4.2 `--exec` フラグ（CP/M → ホスト、テキストのみ）

```bash
# テキストファイルをホストへ取り出す
./build/cpm --exec "TYPE RESULT.TXT" > result.txt

# ASM ソースを取り出す
./build/cpm --exec "TYPE PROG.ASM" > prog.asm
```

- ホスト → CP/M 方向は不可
- バイナリファイルは CONOUT 経由で文字化けするためテキスト専用

### 4.3 R / W コマンド（CP/M 実行中・実装済み）

ディスクイメージに `R.COM` / `W.COM`（カスタムユーティリティ v1.0）が収録されている。
I/O ポート `$A1h` を使ってホストへ通知し、`harness.cpp` の `handle_io` / `read_io` で処理する。

```text
A>R SOURCE.ASM      ← ホスト PC 上のファイルを CP/M ディスクへ
A>W OUTPUT.TXT      ← CP/M ディスクのファイルをホスト PC へ
```

- `cpm` を起動したカレントディレクトリのファイルを参照する
- ファイル名は CP/M 互換形式（英大文字・数字・記号、アンダースコア不可、8+3 文字以内）
- 詳細は [12-CP_M-コマンドリファレンス.md](12-CP_M-コマンドリファレンス.md) 参照

#### R コマンド後に STAT が "1k" と表示される件

R.COM はホストファイルを **128 バイト単位（1 セクタ）** で読み込む。
CP/M のディスク割り当ては **1K 単位（アロケーションブロック = 8 セクタ）** のため、
たとえ数バイトのファイルでも `STAT` は最低 "1k" を表示する。これは正常動作。

```text
ホストファイル 6B → DMA バッファ 128B（6B 実データ + 122B ゼロ埋め）
                 → CP/M ディスク 1K（アロケーションブロック 1 個）
STAT: "Recs 1  Bytes 1k"  ← ファイル内容は正しく保存されている
```

W コマンドで取り出す場合も 128B 単位で書き出されるため、末尾にゼロバイトが付く。
テキストファイルなら `TYPE` で確認すれば実データのみが表示される。

### 4.4 手段の比較

| 手段 | ホスト→CP/M | CP/M→ホスト | 実行中使用 | 追加実装 |
|-----|:-----------:|:-----------:|:----------:|:-------:|
| **cpmtools** | ✅ | ✅ | ❌ 要停止 | 不要 |
| **--exec** | ❌ | ✅ テキストのみ | ❌ | 不要 |
| **R/W コマンド** | ✅ | ✅ | ✅ | 実装済み |

## 5. ビルド手順

```bash
cd examples/06-8080
cmake -B build
cmake --build build
# → build/cpm が生成される
```

依存: Verilator、C++17 コンパイラ。
