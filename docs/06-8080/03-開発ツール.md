# 03. 開発ツール

## 1. アセンブラ

### z80asm（採用）

```bash
sudo apt install z80asm
```

Z80 アセンブラだが、8080 命令セットは Z80 の部分集合なので
Z80 専用命令（JR/DJNZ/EX AF,AF' 等）を使わなければ
生成バイト列は 8080/vm80a で完全動作する。

```bash
z80asm --output=bios.bin bios.asm
```

定数定義は `name: equ 0xHHHH`、ヘキサは `0x` プレフィックス。

### pasmo（代替）

```bash
sudo apt install pasmo
```

`--8080` フラグで 8080 専用ニーモニック（MVI/MOV/JMP 等）を使えるが、
z80asm より情報が少ない。

## 3. CPU 機能テスト

### 8080EX1.COM — Ian Bartholomew

Klaus Dormann の 6502 機能テストに相当する 8080 版。
CP/M 上で動作し、全 8080 命令を網羅的にテストして CRC で検証する。

| 項目 | 内容 |
|------|------|
| 形式 | CP/M `.COM` ファイル |
| 検証方式 | 各命令グループの実行結果を CRC32 で検証 |
| 正解 CRC | ソース内にハードコード済み |
| 入手先 | `https://mdfs.net/Software/Z80/Exerciser/` |

```text
# CP/M 起動後に実行
A>8080EX1
8080 instruction exerciser
...
Tests complete
```

全テストパスで「Tests complete」が表示されれば vm80a が正常動作している証明になる。

### 実行手順

1. `8080EX1.COM` を DSK イメージの A ドライブに配置
2. CP/M を起動
3. `8080EX1` を実行
4. CRC 不一致エラーなく「Tests complete」まで到達することを確認

## 4. ディスクイメージ操作 — cpmtools

```bash
sudo apt install cpmtools
```

DSK イメージへのファイル追加・取り出しに使う。

```bash
# ディレクトリ表示
cpmls -f ibm-3740 cpm22.dsk

# ファイルをイメージに追加
cpmcp -f ibm-3740 cpm22.dsk 8080EX1.COM 0:

# ファイルをイメージから取り出し
cpmcp -f ibm-3740 cpm22.dsk 0:8080EX1.COM .
```

`ibm-3740` フォーマット = 標準 8 インチ SSSD（77T×26S×128B）。
