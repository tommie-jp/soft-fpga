## 2. RTL lint テスト

```bash
cd examples/06-8080
verilator --lint-only \
  verilog/cpm_top.v \
  verilog/vm80a/org/rtl/vm80a.v \
  --top-module cpm_top \
  -Wno-WIDTHEXPAND
```

**ステータス**: ✅ PASS

`-Wno-WIDTHEXPAND` 1 つのみ抑制。`vm80a.v:835` の 1-bit 変数 `cl` を 8-bit ADD に渡す箇所は
第三者 RTL のため修正せず抑制する。`cpm_top.v` 自体の警告はゼロ。

## 5. CP/M ブートテスト

BIOS → CP/M 2.2 → CCP の起動シーケンスを自動検証する。`A>` プロンプトが出れば PASS。

```bash
./build/cpm --boot-test
```

```text
A>
--- CP/M boot: OK (A> found) ---
```

**合格基準**: 標準出力に `A>` が現れ、exit 0 になること。

**ステータス**: ✅ PASS

修正内容: `bios.asm` の `SETTRK` で `ld (CUR_TRK), bc` が z80asm によって
Z80 専用 ED-prefix 命令 (`ED 43 addr`) に変換されていた。
8080 (vm80a) ではこれが未定義動作となり TPA に誤ジャンプしていた。
8080 互換の 2 命令 (`ld a, c; ld (CUR_TRK), a` / `ld a, b; ld (CUR_TRK+1), a`) に置き換えて修正。

## 7. ハーネス sim_test 単体テスト

C++ ハーネス内蔵のスモークテスト。CP/M 環境なしで vm80a 単体動作を確認する。

```bash
cd examples/06-8080
cmake -B build && cmake --build build
./build/cpm --test    # 戻り値 0 = PASS
```

テストプログラム（`harness.cpp` に埋め込み）:

```asm
MVI A, 'H'   ; 0x0000: 3E 48
OUT $01      ; 0x0002: D3 01  (CONOUT)
MVI A, 'i'   ; 0x0004: 3E 69
OUT $01      ; 0x0006: D3 01
HLT          ; 0x0008: 76
```

**合格基準**: `sim_test: 2 bytes out (Hi)` — 2 バイト以上出力で PASS。

**ステータス**: ✅ PASS（修正済み）

修正内容: cpm_top.v の RAM ライトロジックにリセットガード `!reset` を追加し、
Verilator 初期値 `wr_n=0` によるスプリアスライトを防止した。
詳細は [11-vm80a-タイミング解析.md](11-vm80a-タイミング解析.md) 第5節を参照。

## 8. DDT Ctrl+C ウォームブートテスト

DDT 終了後（Ctrl+C）に CP/M が正常にウォームブートし、`A>` プロンプトに戻ることを確認する。

```bash
./build/cpm --ddt-ctrlc-test
```

```text
DDT prompt found
=== output after Ctrl+C (6 chars) ===
^C
A>
```

**合格基準**: Ctrl+C 後の出力が `^C\r\nA>` のみで、ドット (`'.'`) が出力されないこと。

**ステータス**: ✅ PASS

修正内容:

- **根本原因**: WBOOT 時に CCP バイナリを再ロードしないため、CCP 変数 `E3ABh`（Submit フラグ）
  が前回セッションの値（0xFF）を保持し、Submit ファイル処理ループに入った。
- **修正方法**:
  - BIOS WBOOT でポート `0x20` (PORT_WBOOT) に OUT することでハーネスに CCP 再ロードを要求する
  - `harness.cpp` でポート 0x20 OUT を受けると `saved_ccp` を RAM の 0xDC00〜0xE3FF に復元する
  - `saved_ccp` は `sim_init()` 時に cpm22.bin の CCP 部分を保存したもの
  - これにより実機 BIOS の「WBOOT = CCP ディスク再読み込み」と等価の動作になる
