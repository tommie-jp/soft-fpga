# 22. クロック計測とベンチマーク手順

`examples/06-8080/` に置いた BDS C プログラムを CP/M 上でコンパイル・実行する手順。

- **CYCL.C / CYCL.H** — T ステートカウンタ共通ライブラリ（全プログラムに必要）
- **CALIB.C** — カウンタ動作確認（IN 命令 10T × 4 = 40T を実測）
- **SORT.C** — バブルソート vs 挿入ソートの T ステート比較
- **BENCH.C** — IN / OUT / メモリ読み書き / 加算 / 関数呼び出しの計測
- **FREQ.C** — シミュレーション等価クロック周波数を実測し、実機 8080A (2 MHz) と比較

## 1. 前提

### 1.1 起動ディスク

| ドライブ | イメージ | 内容 |
|---------|---------|------|
| A: | `sw/cpm/disks/cpm22.dsk` | CP/M 標準コマンド, R.COM/W.COM |
| B: | `sw/cpm/disks/bdsc.dsk` | BDS C コンパイラ (`CC.COM`, `CLINK.COM`) |

```bash
# examples/06-8080/ から起動する（R コマンドがカレントディレクトリを参照するため）
cd examples/06-8080
./build/cpm
```

### 1.2 ソースファイルの転送

`R.COM` を使って実行中の CP/M に転送する。

```text
A>R CYCL.H
A>R CYCL.C
A>R CALIB.C
A>R SORT.C
A>R BENCH.C
A>R FREQ.C
```

> **別の方法 (cpmtools)**: `cpm` を停止してからホスト側で一括コピーする。
>
> ```bash
> for f in CYCL.H CYCL.C CALIB.C SORT.C BENCH.C FREQ.C; do
>   cpmcp -f ibm-3740 sw/cpm/disks/cpm22.dsk $f 0:$f
> done
> ```

## 2. CALIB.C — カウンタ動作確認

### 2.1 コンパイル

```text
A>B:CC CYCL.C
A>B:CC CALIB.C
A>B:CLINK CALIB CYCL
```

> `CYCL.CRL` は 1 回コンパイルすれば使い回せる。SORT.C / BENCH.C でも同じ CYCL.CRL を使う。

### 2.2 実行

```text
A>CALIB
```

### 2.3 期待される出力

```text
=== Cycle Counter Calibration ===

cycl_read overhead : ~70 T
empty loop 500x    : ~12500 T  (~25 T/iter)
4x IN loop 500x    : ~32500 T  (~65 T/iter, +40 vs empty)
  => +40 T vs empty (~40 expected)
```

**判定**:

- `+40 T` → IN 命令 = 10T / 本 × 4 本 → カウンタ正常
- `BDS C for ループ` 自体は約 25 T/iter（LXI + DCR + JNZ + INX 相当）

## 3. SORT.C — バブルソート vs 挿入ソート

### 3.1 コンパイル

```text
A>B:CC SORT.C
A>B:CLINK SORT CYCL
```

（`CYCL.CRL` は § 2.1 でコンパイル済みのものを再利用）

### 3.2 実行

```text
A>SORT
```

### 3.3 期待される出力

```text
=== Sort Benchmark  N=32 ===

Worst case (reverse):
  bubble sort             : 約 30000 T
  insertion sort          : 約 20000 T

Random data:
  bubble sort             : 約 20000 T
  insertion sort          : 約 13000 T
```

**読み方**:

- 逆順（最悪ケース）で挿入ソートが約 35% 速い
- 8080 では swap = 3 × `MOV` = 15T、シフト = 1 × `MOV` = 7T のため

## 4. BENCH.C — I/O・メモリ・演算計測

### 4.1 コンパイル

```text
A>B:CC BENCH.C
A>B:CLINK BENCH CYCL
```

### 4.2 実行

```text
A>BENCH
```

### 4.3 期待される出力

```text
=== 8080 Benchmark  N=500 ===

Loop base: 25 T/iter

  4x IN           : 40 T extra (exp 40)
  4x OUT          : 40 T extra (exp 40)
  4x mem rd       : ~28 T extra (exp ~28)
  4x mem wr       : ~28 T extra (exp ~28)
  8x int add      : ~32 T extra (exp ~32)
  C func call     : ~27 T extra (exp ~27)
```

**読み方**: "exp" と実測の差が ±2T 以内なら vm80a RTL の T ステート実装が正確。

| 計測対象 | 命令 | 期待値 |
|---------|------|--------|
| `inp()` × 4 | `IN p` × 4 | 40T (= 10T × 4) |
| `outp()` × 4 | `OUT p` × 4 | 40T |
| `char` 読み × 4 | `MOV A,M` × 4 | 28T (= 7T × 4) |
| `char` 書き × 4 | `MOV M,A` × 4 | 28T |
| `int` 加算 × 8 | `ADD r` × 8 | 32T (= 4T × 8) |
| C 関数呼び出し | `CALL nn + RET` | 27T (= 17T + 10T) |

## 5. FREQ.C — シミュレーション等価クロック周波数計測

### 5.1 コンパイル

```text
A>B:CC FREQ.C
A>B:CLINK FREQ CYCL
```

（`CYCL.CRL` は § 2.1 でコンパイル済みのものを再利用）

### 5.2 実行

```text
A>FREQ
```

### 5.3 期待される出力

```text
=== Equivalent Clock Frequency Ver. 0.1.2 ===
  Measuring : 5000-iter loop (~125 kT)

  Simulated :    8,183,192 Hz
  Real 8080A :    2,000,000 Hz
  vs 8080A  : 409%
```

**読み方**:

- `Simulated` — ウォールクロック（`FSTA` / `FLAT` ポート）と T ステートカウンタから算出した等価クロック周波数
- `Real 8080A` — 実機比較基準（2 MHz 固定）
- `vs 8080A` — `Simulated / 2 MHz × 100`（Verilator はホスト CPU 速度に依存するため数百〜数千 % になる）

**計測原理**:

```c
freq_start();          // OUT 34h: ウォールクロック + サイクルカウンタを記録
for (i = 0; i < N; i++) ;  // N=5000 の空ループ
freq_latch();          // OUT 38h: 経過時間から Hz を確定
mhz = freq_mhz();     // IN 34h/35h
khz = freq_khz();     // IN 36h
hz  = freq_hz();      // IN 37h
```

> `vs 8080A` は `int` 16 ビットの上限を避けるため MHz > 30 の場合は 30,000 kHz に丸めて計算している。

## 6. まとめコマンド列

全プログラムを一括でコンパイルして実行する場合:

```text
A>R CYCL.H
A>R CYCL.C
A>R CALIB.C
A>R SORT.C
A>R BENCH.C
A>R FREQ.C
A>B:CC CYCL.C
A>B:CC CALIB.C
A>B:CC SORT.C
A>B:CC BENCH.C
A>B:CC FREQ.C
A>B:CLINK CALIB CYCL
A>B:CLINK SORT CYCL
A>B:CLINK BENCH CYCL
A>B:CLINK FREQ CYCL
A>CALIB
A>SORT
A>BENCH
A>FREQ
```

## 7. BDS C の制約（注意点）

| 制約 | 内容 |
|------|------|
| `long` 非対応 | `int` 16 ビットのみ → 計測上限 65535 T |
| 識別子は 8 文字有効 | `measure_overhead` と `measure_empty` が衝突するため `do_ovhd` / `do_empty` に改名済み |
| `#asm`/`#endasm` 非対応 | NOP の代わりに `inp(0x10)` × 4 で 40T を挿入 |
| コンパイル拡張子 | `.C` → `.CRL`（中間）→ `CLINK` → `.COM` |
